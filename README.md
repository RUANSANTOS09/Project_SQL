## 📖 Sobre este repositório

Este repositório reúne projetos práticos que combinam **SQL** e **Python**, desenvolvidos como parte de uma jornada de aprendizado contínuo em **Engenharia de Dados**.

Cada projeto aqui dentro resolve um problema específico — geralmente inspirado em cenários reais de extração, filtragem e geração de relatórios sobre dados — utilizando consultas SQL para acessar o banco de dados e Python para processar, transformar e apresentar os resultados.

A proposta não é apresentar exemplos isolados de sintaxe, mas sim **simular tarefas reais** do dia a dia de quem trabalha com dados: conectar a um banco, escrever consultas com propósito, tratar os resultados em código e entregar uma saída útil — seja um relatório, uma validação ou uma análise simples.

---

## 🧭 Como este repositório evolui

Os projetos aqui não seguem um nível fixo de dificuldade — eles **acompanham o avanço do aprendizado**. Por isso, é esperado que:

- Projetos mais recentes utilizem conceitos mais avançados de SQL (como `JOIN`, subconsultas, funções de agregação) e de Python (como tratamento de exceções, modularização, boas práticas de segurança)
- A qualidade e complexidade dos projetos aumentem com o tempo, refletindo o progresso real do aprendizado
- Nem todo projeto utiliza as mesmas ferramentas ou bibliotecas — cada um é independente, com seu próprio escopo

Cada projeto possui sua própria documentação interna, descrevendo o cenário proposto, os conceitos aplicados e como executá-lo.

---

## 🛠️ Tecnologias utilizadas

- **SQL** — consultas sobre banco de dados relacional (MySQL)
- **Python 3** — processamento, automação e geração de relatórios
- **mysql-connector-python** — integração entre Python e o banco de dados

## ▶️ Como navegar

Cada projeto está organizado em sua própria pasta, contendo:

```
.
├── nome-do-projeto/
│   ├── main.py          # ou script principal
│   ├── README.md          # documentação específica do projeto
│   └── (arquivos de saída gerados pela execução)
```

Para executar qualquer projeto, acesse a pasta correspondente e siga as instruções do README específico contido nela.
